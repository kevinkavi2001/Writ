### One nuance worth flagging

`--playbook-compliance` reported `compliant=0` with skip_points
"testing, implementation". Investigation: my Phase 6i emit fires
`step_index = phases.index(new_phase)` -- so the first advance
(planning -> testing) fires at step_index=1, NOT step_index=0.
The analyzer counts a run "compliant" only if the sequence forms
0, 1, 2, ... -- so a run starting at 1 is technically non-compliant
by its definition. The data is correct; the labeling is misleading.

This is a small design tension between Phase 6i's "step_index =
ordinal of new phase" mapping and Phase 5's "compliant = sequence
starts at 0" definition. Either the emit should also fire on
session start (planning at index 0), or the analyzer should accept
runs that start at index 1. Worth a small follow-up but does NOT
block Phase 6 closure -- the analyzer is producing real signal,
just with a stricter compliance bar than the producer intended.
