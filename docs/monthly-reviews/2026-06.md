## Decisions taken this month

- Phase 6j (methodology integration verification) shipped via the multi-part change in commit-pending: `node_types` opt-in on `QueryRequest`, methodology-companion query in `writ-rag-inject.sh`, and live Neo4j migration of the methodology corpus.
- Hook stderr surfacing (Part B): mutating `_writ_session update` calls in `writ-rag-inject.sh` and `writ-posttool-rag.sh` now redirect stderr to `${WRIT_HOOK_LOG:-/tmp/writ-hooks.log}` instead of `/dev/null`. The legacy-cache KeyError that motivated this would now produce visible log output.
- Always-on independent budget (Part C): `writ-session.py` schema gained `always_on_budget` (default 5000 from `budget.json:always_on_cap`) and `always_on_tokens_used`, plus a `--add-always-on-tokens N` flag wired into the always-on injection path. Cumulative cost now visible via `event: always_on_inject` friction events.
- Phase 6e/6f/6g status correction: those sub-phases were marked "verified-shipped via promotion" but only file-system promotion happened. Live Neo4j ingestion was missing until this session ran `scripts/migrate.py --methodology-dir bible/methodology` (added 50 new-type + 10 methodology Rule nodes; 120 edges).
